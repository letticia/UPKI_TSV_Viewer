# 電子証明書自動発行支援システム から取得したTSVファイルの内容を表示し、また絞り込めるWebアプリケーションを作成

## 元になるTSVファイル
- 同ディレクトリ内にある、拡張子「.tsv」のファイル
- 複数ある場合は、作成日の新しいものを使用する
- 1行目は、各列の内容を表すラベルである
- 列数が20の場合は、ACME情報一括ダウンロードから取得したファイルである
- 列数が46以上の場合は、サーバ証明書またはクライアント証明書の情報一括ダウンロードから取得したファイルである

## アプリケーションの要件
- 配色は、現在のものに加えて、標準的な白系のライトテーマ、明るい環境での作業に適したものを選択できるようにする。デフォルトは白系とする。
- 元になるTSVファイルの内容を表形式で一覧表示できる。
- 現在表示しているTSVファイルが、ACME情報一括ダウンロードから取得したファイルか、サーバ証明書またはクライアント証明書の情報一括ダウンロードから取得したファイルかを判別し、アプリケーションのタイトルに表示する。条件は以下の通り。
  - ACME情報一括ダウンロードから取得したファイルの場合、「ACME情報ビューア」 とする。
  - サーバ証明書またはクライアント証明書の情報一括ダウンロードから取得したファイルの場合、「サーバ証明書・クライアント証明書情報ビューア」とする。
- どの項目でも検索が可能である。検索ボックスの右横に「検索」ボタンを設置する。
- 検索結果は、絞り込まれた行のみを表示する
- 表示する列もユーザーが任意で選択できるようにする。表示/非表示を切り替えれるようにする。ボタンに「表示/非表示」と表示する。
- 検索ボックスの直下に、絞り込み用のボタンを設置する。
  - 検索条件をリセットするボタン
  - 検索結果をエクスポートするボタン
  - 「最終更新日」列の年月日をもとに、「今日から1週間以内」「今日から1ヶ月以内」「今日から3ヶ月以内」を絞り込むボタン
  - 「発行・更新申請-証明書有効期限」列の年月日をもとに、残り有効期間が「今日から1週間以内」「今日から30日以内」を絞り込むボタン
- dNSName 列は、FQDNごとにカンマで区切られている。改行を入れて表示する。
- 「CSR」列、「発行・更新申請-証明書」列は内容が多いので、非表示とする。表示されている列をクリックしたら内容を表示するようにする。
- いくつかの列では、ステイタスをあらわす数字が入力されている。その数字の意味を、「値：意味」 の形で置き換えて表示する。対応表は後述する。

# 列名ごとの対応

## ACME情報一括ダウンロードから取得した場合

### 【申請種別】
1:ACME作成
2:ACME更新
3:ACME削除

### 【状態】
1:ACME有効
2:ACME無効  
-1:フォーマットエラー
-2:ACME連携処理エラー

### 【証明書プロファイルID】
3:サーバ証明書（sha256WithRSAEncryption）
11:サーバ証明書（ecdsa-with-SHA384）

### 【停止理由コード】
0:unspecified(未定義)
1:keyCompromise(秘密鍵の漏洩による証明書の脆弱化)
3:affiliationChanged(主体の所属変更)
4:superseded(証明書の更新)
5:cessationOfOperation(証明書の運用停止)

## サーバ証明書またはクライアント証明書の情報一括ダウンロードから取得した場合

### 【証明書プロファイルID】
1:サーバ証明書（sha1WithRSAEncryption）（有効期限：2016年12月31日まで）
2:サーバ証明書（sha1WithRSAEncryption）（有効期限：2015年12月31日まで）
3:サーバ証明書（sha256WithRSAEncryption）
4:クライアント証明書（sha1WithRSAEncryption）（有効期限：2016年12月31日まで）
5:クライアント証明書（sha256WithRSAEncryption）(証明書有効期間:52ヶ月)
6:S/MIME証明書（sha1WithRSAEncryption）（有効期限：2016年12月31日まで）
7:S/MIME証明書（sha256WithRSAEncryption）(2022年3月22日以降は証明書有効期間:823日。それ以前は証明書有効期間:52ヶ月)
8:コード署名用証明書（sha1WithRSAEncryption）（有効期限：2016年12月31日まで）
9:コード署名用証明書（sha256WithRSAEncryption）
11:サーバ証明書（ecdsa-with-SHA384）
13:クライアント証明書（sha256WithRSAEncryption）(証明書有効期間:13ヶ月)
14:クライアント証明書（sha256WithRSAEncryption）(証明書有効期間:25ヶ月)
15:S/MIME証明書（sha256WithRSAEncryption）(証明書有効期間:13ヶ月)
16:S/MIME証明書（sha256WithRSAEncryption）(証明書有効期間:25ヶ月)

### 【状態】
0:証明書発行処理中 
1:証明書更新処理中 
2:証明書失効処理中 
3:発行案内メール送信待ち 
4:発行案内メール送信済み 
5:証明書発行処理完了 
6:証明書更新処理完了 
7:証明書失効処理完了 
8:新証明書への置き換え待ち 
9:発行・更新完了メール送信待ち 
10:失効完了メール送信待ち 
-1:フォーマットエラー 
-2:認証局連携API処理エラー 
-3:支援システム実行エラー

### 【失効申請-失効理由コード】
0:unspecified(未定義) 
1:keyCompromise(秘密鍵の漏洩) 
3:affiliationChanged(利用管理者DNの変更) 
4:superseded(証明書の更新) 
5:cessationOfOperation(証明書の運用停止)

### 【ダウンロード方法】
1:P12個別
2:P12一括
3:ブラウザ発行
4:CSR発行
